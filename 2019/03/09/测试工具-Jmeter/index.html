JMeter的基本介绍和入门

介绍

JMeter 是 Apache 组织的开源的项目，主要能做功能测试和性能测试，能够对 HTTP 和 FTP 服务器进行压力和性能测试。
测试环境搭建
使用 JMeter 需要有 java 环境，然后通过在官网 http://jmeter.apache.org/download_jmeter.cgi 下载 JMeter 。
(1)下载

（2）windows 下启动，先解压，然后通过 bin 中的 jmeter.bat 启动 jmeter gui 界面。关闭命令行即退出 jmeter


测试实例

1、在 TestPlan 中添加线程组，JMeter是由Java实现的，并且使用一个Java线程来模拟一个用户，因此线程组（Thread Group）就是指一组用户的意思，换句话说一个线程组就是一组虚拟用户（virtual users），这些虚拟用户用来模拟访问被测系统。

（1）Number of Threads (Users) ：当前线程组中每个线程的模拟个数
（2）Ramp-Up Period (in seconds) ：虚拟用户增长市场，以秒为单位。举个例子，这里设置成300秒，如果“线程数”输入为100，则意味着在5分钟内100虚拟用户测试完毕。
（3）Loop Count ：循环的次数，填 1 则只执行一次，勾选 Forever 则永远停不下来了。
（4）Scheduler：调度器，可以在里面设置测试的时长

2、可以通过在 TestPlan 中添加 User Defined Variables 来用户定义的全局变量，同样在线程组也可以通过定义 User Defined Variables 来定义线程组中的变量。
同样也可以通过线程组来定义每一个线程 HTTP 的头部信息或者 Cookie 等信息。

3、在 ThreadGroup 中添加 HTTP request 请求。

可以通过 HTTP Request 中的 method 来定义是使用什么方法来请求地址，Path 为请求的地址，如果在用户变量中有设置地址则可以通过 ${变量名} 来使用（输入框只要能输入，基本上都可以通过 ${变量名} 来获取变量的值）。Parameters 为变量提交与 Postman 中的 Params 基本一直，Body Data 中可以传输 json 格式的数据。Files Upload 是为文件上传。

4、可以通过在 HTTP Request 中添加 response assertion 来添加断言。

Response Assertion ,响应断言，主要有响应状态码和响应时间，响应返回内容等。这里的断言方式为对响应码进行校准。下面的 Custom failure message 可以输入当出错时输出的信息。同样可以通过 ${} 来输出执行的变量。
